HiCuPP Release July 4th, 2002

Author: Bart Jacobs.

New In This Release

This release fixes a problem with the node popup menu when the program is run in Windows XP. In the previous release, a java.awt.PopupMenu object is created and added to the tree document after the user clicks the mouse on the node. In the current release, the PopupMenu object is created and added to the tree document once when the TreeDocument is constructed.

Structure Of The Source Code

Package Jama

Jama is a library for manipulating matrices which I downloaded from the Internet. It is used by classes in the hicupp package for computing the split axis.

Package imageformats

These classes are used by the hicupp and interactivehicupp packages to load the images selected for segmentation by the user.

Package hicupp

Contains the classes that actually compute the split axis. The main class for the computation is hicupp.Clusterer. This package also contains the user interface for sequential hicupp.

The projection indices are defined in hicupp.ProjectionIndexFunction. It should be easy to add new projection indices there. When done correctly, new projection indices automatically show up in the user interface.

The hicupp.FunctionMaximizer class implements the simplex algorithm.

Package interactivehicupp

The structure of this package is somewhat complicated. This is mainly due to the effort to share as much code as possible between Hicupp for Images and Hicupp for General Point Sets.

The DocumentXXX classes implement generic document management: opening and saving documents. The TreeDocumentXXX classes instantiate the general document concept for so-called tree documents, which are the hierarchical clusterings. A tree document consists of a number of nodes and a number of splits. There is one root node, and each node has zero or one split. Each split has a left child node and a right child node.

The main difference between HiCuPP for Images and HiCuPP for General Point Sets is where the points come from: in the former case, they come from bitmap files, and in the latter case, they come from ASCII files. Therefore, I have factored out the concept of PointsSource. There is an ImagePointsSourceProvider and a GeneralPointsSourceProvider. Since there is a need for two-way communication between the PointsSourceProvider and the TreeDocument, the TreeDocument implements the PointsSourceClient interface.
